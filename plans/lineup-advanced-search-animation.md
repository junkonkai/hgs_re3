# 詳細検索の開閉アニメーション案（高さに依存しない／中身に合わせる）

## 課題
- `max-height: 0 → 1000px` だと「魔法の数字」が必要で、初回だけ中身のレイアウトが遅れて表示される問題があった。
- 高さを固定せず、中身の高さに合わせてスムーズに開閉したい。

## 案

### 1. CSS Grid の `0fr` / `1fr`（採用）
- ラッパーを `display: grid`、閉じているときは `grid-template-rows: 0fr`、開いているときは `1fr`。
- 子要素に `min-height: 0` と `overflow: hidden` を指定し、0fr のときは行の高さが 0 になり中身がクリップされる。
- **メリット**: 固定の高さ不要。中身の高さで開く。初回からレイアウト済みなので表示遅れなし。CSS のみ。
- **デメリット**: 内側に 1 段ラッパー（`.advanced-search-wrapper__inner`）が必要。

### 2. JS で `height: 0` → `height: scrollHeight`
- 開くときに `scrollHeight` を測って `height` を指定し、transition でアニメーション。
- **メリット**: 見た目は自然。**デメリット**: 計測のため一時的に表示するなどの配慮が必要で、JS が増える。

### 3. `opacity` + `visibility` のみ
- 閉じているときは `opacity: 0; visibility: hidden`、開くと `opacity: 1; visibility: visible`。
- **デメリット**: 閉じていてもレイアウト上は高さを取るため「閉じた状態で高さ 0」にはならない。

### 4. `clip-path` で縦方向にクリップ
- 例: `clip-path: inset(0 0 100% 0)` → `inset(0 0 0 0)`。
- **デメリット**: `clip-path` はレイアウトの高さを変えないため、閉じた状態で高さ 0 にするには結局ラッパーの高さ制御が必要。

---

**結論**: 案 1（Grid 0fr/1fr）を採用し、初期表示は閉じた状態に戻す。
