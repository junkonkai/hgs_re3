# フランチャイズ一覧の今風リニューアル案

## 目標

- 「あかさたな」起点の探索しかできない状態から、`検索 + フィルタ + 段階的読込`で目的のフランチャイズに最短到達できる体験へ変更する。
- 既存導線（`Game.Franchises`）と詳細ページ遷移は維持し、破壊的変更を避ける。

## 現状の課題（コード上）

- [GameController.php](/var/www/html/hgs_re3/app/Http/Controllers/GameController.php) の `franchises()` で、全prefix分を毎回取得しており、表示対象外も含めてクエリ実行される。
- [franchises.blade.php](/var/www/html/hgs_re3/resources/views/game/franchises.blade.php) はprefix単位のアコーディオン前提で、文字入力での絞り込みや並び替えができない。
- ルートは [web.php](/var/www/html/hgs_re3/routes/web.php) の `/franchises/{prefix?}` でprefix前提のURL構造。

## 推奨アプローチ（優先）

- **Step 1: 検索ファーストUIを追加（互換維持）**
  - フランチャイズ一覧上部に検索入力（名前/よみ検索）を追加。
  - 初期表示は「人気順 or 更新順 + ページネーション」、必要時のみ検索。
  - prefixナビは「補助フィルタ」として残す（完全削除はしない）。
- **Step 2: サーバー側をクエリパラメータ化**
  - `?q=&initial=&rating=&sort=&page=` を受ける一覧API的なコントローラ処理に寄せる。
  - `paginate()` を使って段階読込にし、初回負荷を下げる。
- **Step 3: 将来拡張**
  - タイトル検索で使っている全文検索基盤（Meilisearch）と同様に、フランチャイズも全文検索対象にする。
  - サジェスト（インクリメンタル検索）を導入し、一覧スクロールを減らす。

## 実装ポイント（最小差分で始める）

- [GameController.php](/var/www/html/hgs_re3/app/Http/Controllers/GameController.php)
  - `franchises()` に `q` と `initial` の受け口を追加。
  - `orWhere` の固定配列ループ中心から、条件付きクエリビルド + `orderBy` + `paginate` へ変更。
- [franchises.blade.php](/var/www/html/hgs_re3/resources/views/game/franchises.blade.php)
  - 検索フォーム、フィルタUI、件数表示、ページャを追加。
  - 既存のprefixアコーディオンは「初期文字フィルタ」として簡略化して共存。
- [web.php](/var/www/html/hgs_re3/routes/web.php)
  - 既存ルートは維持しつつ、prefixは任意の互換パラメータ扱いへ（将来はquery param中心へ誘導）。

## UIの方向性（今風）

- 一覧は「カード/リスト + 先頭固定検索バー」。
- 絞り込みは chips（例: 初期文字、R18表示可否、並び順）。
- モバイル優先で、タップ1回で詳細へ遷移できる密度に調整。

## 段階移行プラン

- Phase A（低リスク）: 現ルート互換のまま検索・ページング追加。
- Phase B（改善）: prefix中心UIを縮小し、検索中心レイアウトへ。
- Phase C（拡張）: サジェスト/全文検索連携を導入。

## 完了条件

- フランチャイズ一覧で「名前の一部検索」「初期文字フィルタ」「ページング」が機能する。
- 既存の `Game.Franchises` から詳細遷移が壊れない。
- 体感性能（初回表示件数、クエリ数）が現状より改善する。
